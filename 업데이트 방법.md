배포는 태그 push 기준으로 진행됩니다.

순서:

1. `package.json` 버전 변경
2. `package-lock.json` 버전 같이 변경
3. 커밋
4. 같은 버전으로 태그 생성
5. `main`과 태그 push

예시:

```bash
git add .
git commit -m "release: v1.0.5"
git tag v1.0.5
git push origin main
git push origin v1.0.5
```

정상 동작하면 GitHub Actions가 실행되면서 아래 파일이 Release에 올라갑니다.

- `NICEENTECH-Setup-1.0.5.exe`
- `NICEENTECH-Setup-1.0.5.exe.blockmap`
- `latest.yml`

주의:

- 태그 버전과 `package.json` 버전이 다르면 Release 업로드 실패
- 자동업데이트는 `latest.yml` 까지 같이 올라가야 동작
- 이미 같은 태그가 있으면 새 버전으로 다시 태그를 만들어야 함
